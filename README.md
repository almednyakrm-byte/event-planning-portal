# Event Planning Portal
## Overview
The Event Planning Portal is a web-based application designed to simplify the process of planning and managing events. The purpose of this project is to provide a user-friendly platform for event organizers to create, manage, and track events, as well as for attendees to register and engage with events.

## Project Structure
The project is structured into the following components:
* `app`: The main application directory containing the backend logic.
* `database`: The database schema and migration scripts.
* `docker`: The Docker configuration files for containerization.
* `frontend`: The client-side application directory containing the user interface.
* `tests`: The unit and integration tests for the application.

## Running the Environment
To run the environment using `docker-compose up`, follow these steps:
1. Clone the repository to your local machine.
2. Navigate to the project directory in your terminal.
3. Run the command `docker-compose build` to build the Docker images.
4. Run the command `docker-compose up` to start the containers.
5. Access the application at `http://localhost:8000` in your web browser.

## Modules, Tables, and Roles
The application consists of the following modules:
* **Event Management**: Create, edit, and delete events.
* **Registration Management**: Manage attendee registrations and payments.
* **User Management**: Manage user accounts and roles.
The database schema includes the following tables:
* **events**: Stores event information.
* **registrations**: Stores attendee registration information.
* **users**: Stores user account information.
The application has the following roles:
* **Admin**: Has full access to all features and settings.
* **Organizer**: Can create and manage events, but cannot access user accounts or settings.
* **Attendee**: Can register for events and view event information.
